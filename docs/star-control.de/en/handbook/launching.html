Handbook

    Launching the Game

    Configure performance options and start Star Citizen with a single click.

    Launch Control

    The large launch button at the top starts Star Citizen. While the game is running, the button changes to a stop button that lets you terminate the process. An info card below shows the currently active Wine runner and prefix path.

    Performance Options

    Toggle these options to optimize Star Citizen's performance on your system:

    ESync

    Eventfd-based synchronization for Wine. Reduces CPU overhead for synchronization primitives. Recommended to keep enabled for better performance.

    FSync

    Futex-based synchronization (requires kernel 5.16+). More efficient than ESync when supported. If your kernel supports it, enable both ESync and FSync.

    DXVK Async

    Enables asynchronous shader compilation in DXVK. Reduces stuttering caused by shader compilation during gameplay. Highly recommended.

    HDR

    Enables High Dynamic Range output. Requires a HDR-capable monitor and compositor support (e.g., KDE Plasma 6 with HDR enabled).

    FSR (FidelityFX Super Resolution)

    AMD's upscaling technology. Renders the game at a lower resolution and upscales for better performance while maintaining visual quality.

    MangoHUD

    Displays a performance monitoring overlay showing FPS, CPU/GPU usage, temperatures, and frame times. Useful for troubleshooting performance issues.

    DXVK HUD

    Shows DXVK debug information overlay. Useful for developers and advanced users to diagnose rendering issues.

    Wayland Support

    Star Control includes experimental Wayland support for native Wayland compositors.

    • Enable Wayland - Toggle to use Wayland instead of XWayland. Automatically disabled if fractional scaling is detected (which is currently incompatible).
    • Primary Monitor - Select which monitor to use for the game. Star Control auto-detects connected monitors with their resolutions.
    Wayland support is experimental. If you experience issues, disable it to fall back to XWayland. Fractional scaling (e.g., 125%, 150%) is not compatible with native Wayland mode.

    Custom Environment Variables

    For advanced users, you can define custom environment variables that will be set when launching Star Citizen. Each variable has:

    • An enable/disable toggle to quickly activate or deactivate without deleting
    • A KEY field (automatically converted to uppercase, only A-Z, 0-9, and underscore allowed)
    • A VALUE field

    A conflict warning badge appears if a custom variable would override one of the built-in options (like ESync or FSync).

    Log Output

    The log panel at the bottom shows real-time output from the game launch process. It streams log lines as they arrive and auto-scrolls to the latest entry. This is helpful for diagnosing launch failures or monitoring the startup sequence.

    All launch options are disabled while the game is running to prevent configuration changes during gameplay. They become editable again once the game exits.
