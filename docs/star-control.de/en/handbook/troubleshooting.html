Handbook

    Troubleshooting & FAQ

    Solutions to common problems, log file locations, and known limitations.

    Log File Locations

    When something goes wrong, these log files contain diagnostic information:

    • Star Control application log - ~/.config/star-control/logs/
    • Star Control debug log - ~/.config/star-control/logs/debug.log
    • Wine/Game output - Shown in the Launch page log panel during gameplay

    Set the log level to Debug in Settings for the most detailed output when diagnosing issues.

    Common Problems

    Game won't launch

    • Verify a Wine runner is selected and installed (check the Wine Runners page)
    • Run the system check on the Installation page to verify prerequisites
    • Check the Launch page log output for error messages
    • Try a different Wine runner - some versions work better than others

    System check failures

    • Inotify mapcount too low - Set vm.max_map_count=16777216 in /etc/sysctl.d/ and run sudo sysctl --system. See the LUG Wiki for distribution-specific instructions.
    • File descriptor limit too low - Add * soft nofile 524288 and * hard nofile 524288 to /etc/security/limits.conf and restart your session.
    • No Vulkan support - Install the appropriate Vulkan driver for your GPU (mesa-vulkan-drivers for AMD/Intel, nvidia-vulkan-icd for NVIDIA).

    Poor performance

    • Enable ESync and FSync on the Launch page
    • Enable DXVK Async to reduce shader compilation stuttering
    • Use the USER.cfg editor to lower graphics settings
    • Enable MangoHUD to monitor CPU/GPU usage and identify bottlenecks
    • Ensure DXVK is installed and up to date (check Wine Runners)

    Keybindings on wrong device

    Linux joystick instance numbers can differ from what Star Citizen expects. Use the Keybindings page to reorder your joystick instances via drag and drop.

    Wayland issues

    • Wayland support is experimental. If you experience problems, disable it on the Launch page to fall back to XWayland.
    • Fractional scaling (125%, 150%) is not compatible with native Wayland mode.

    GitHub API rate limit

    If you see rate limit errors when browsing Wine runners or checking for updates, set a GitHub personal access token in Settings. This increases the limit from 60 to 5,000 requests per hour.

    Known Limitations

    • Star Control is in early development. Some features may be incomplete or change between versions.
    • The application currently supports English and German. Other UI languages are not yet available.
    • Star Citizen's anti-cheat (EasyAntiCheat) must be compatible with Wine/Proton for the game to run.
    • Some Wine runners may work better than others depending on your hardware and distribution.

    Getting Help

    If you encounter an issue not covered here:

    • Check the GitHub Issues page for known bugs and feature requests
    • Open a new issue with your log files and system information
    • Visit the Star Citizen LUG Wiki for general Linux-related Star Citizen help
